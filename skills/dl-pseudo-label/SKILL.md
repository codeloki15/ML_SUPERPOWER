# Pseudo Label

Generate pseudo-labels on unlabeled data via a trained model, filter by confidence, retrain (or distill) on the augmented dataset. K-fold-safe by design — pseudo-labels for fold K are generated by a model trained without fold K.

## When to invoke

- User has a small labeled set + larger unlabeled pool (semi-supervised setting).
- User asks "pseudo-label", "self-training", "semi-supervised", "use unlabeled data".
- After a baseline model is trained AND `dl-cv-eval-*` or `dl-nlp-eval-*` shows reasonable headline metric (>random baseline) — you need a competent labeler before pseudo-labeling helps.

## When NOT to invoke

- Without a held-out true-labeled validation set. Pseudo-labels can degrade quality silently; you need a real eval to detect.
- Extreme label noise tasks (>30% noisy labels in original training). Pseudo-labels will inherit and amplify the noise.
- When getting more real labels is feasible — pseudo-labels are a band-aid, not a cure.
- Multi-label tasks without per-label confidence thresholds (single threshold on multi-label is wrong).

## Decision rules

### Confidence threshold

- Default: 0.9 for hard labels (multi-class). Raise to 0.95+ if pseudo-label quality on a small sanity-check is poor.
- For binary classification: 0.85 if balanced, raise on the minority class.
- For NER: 0.95 per token; require ALL tokens in a span above threshold to accept the span.
- For regression: confidence-based filtering is harder; use ensemble variance as a proxy (drop predictions with high variance).

NEVER lower below the model's calibration ECE. If ECE=0.10, threshold of 0.9 actually accepts predictions that are 80% accurate at best.

### K-fold safety

The MOST common pseudo-label bug: train fold-K model, generate pseudo-labels using fold-K model, train final model on (real + pseudo) including pseudo-labels for fold-K's data → leak.

CORRECT pattern:
1. Train K fold-models (K-fold cross-validation).
2. For each unlabeled example, generate K pseudo-labels (one per fold-model).
3. Average / vote / select the most-confident.
4. The pseudo-label is K-fold-safe because no single fold-model both saw and labeled its own held-out data.

### Pseudo-label use

- **Hard labels** (one-hot, argmax of softmax): standard. Use when filter threshold is high (>0.9).
- **Soft labels** (full softmax distribution): better for distillation. Use when handing off to `dl-distillation`.

## Process

### Step 1 — Prerequisite check

Verify a baseline model is trained AND beats trivial baseline by a reasonable margin. If not, fix the baseline first; pseudo-labeling can't rescue a broken model.

### Step 2 — Inspect calibration

Read the model's ECE from `<workdir>/metrics.json` (Phase 2 eval skills include this). Pick threshold above ECE; default 0.9.

### Step 3 — Generate pseudo-labels

Run inference on the unlabeled pool. For K-fold setup, run K fold-models in parallel. Save logits to `<workdir>/pseudo_labels/raw_logits_<fold>.npz`.

### Step 4 — Filter by confidence

Apply threshold. Save accepted pseudo-labels to `<workdir>/pseudo_labels/accepted.csv` with columns `path/text, pseudo_label, confidence`.

Surface counts: how many of the unlabeled pool were accepted (typical: 30-60%; if <10% pseudo-labels are useless, if >80% the threshold is too low).

### Step 5 — Inspect quality on a sanity sample

Pick 20 random accepted pseudo-labels. Manually inspect — are they correct? If <80% look correct, raise threshold and re-filter.

### Step 6 — Train on (real + pseudo)

Re-invoke training (Phase 2 skills) with the augmented dataset. Hand off to `dl-distillation` if doing distill-into-single-model afterward.

### Step 7 — Eval

Compare new model's metric to baseline. If pseudo-labels helped, the metric should improve; if it degraded, threshold was too low or labels too noisy.

### `<workdir>/src/_pseudo_label.py`

```python
"""K-fold-safe pseudo-labeling pipeline."""
import json
import os
from pathlib import Path

import numpy as np

WORKDIR = Path(os.environ.get("WORKDIR", ".")).resolve()


def generate_pseudo_labels(fold_models: list, unlabeled_loader, num_classes: int) -> np.ndarray:
    """For each unlabeled example, run all K fold-models. Returns (N, K, C) probability tensor."""
    import torch
    import torch.nn.functional as F

    all_probs = []
    for fold_idx, model in enumerate(fold_models):
        model.eval()
        device = next(model.parameters()).device
        batch_probs = []
        with torch.no_grad():
            for batch in unlabeled_loader:
                if isinstance(batch, dict):
                    batch = {k: v.to(device) if hasattr(v, "to") else v for k, v in batch.items()}
                    logits = model(**batch).logits
                else:
                    inputs = batch[0].to(device) if isinstance(batch, (list, tuple)) else batch.to(device)
                    logits = model(inputs)
                probs = F.softmax(logits, dim=-1)
                batch_probs.append(probs.cpu().numpy())
        all_probs.append(np.concatenate(batch_probs, axis=0))
    return np.stack(all_probs, axis=1)  # shape (N, K, C)


def filter_by_confidence(probs_NKC: np.ndarray, threshold: float = 0.9, mode: str = "average") -> tuple:
    """probs_NKC: (N, K, C). Returns (accepted_indices, hard_labels, mean_confidence)."""
    if mode == "average":
        ensemble = probs_NKC.mean(axis=1)  # average over folds
    elif mode == "vote":
        # majority vote over folds
        per_fold_argmax = probs_NKC.argmax(axis=-1)  # (N, K)
        from scipy.stats import mode as scipy_mode
        ensemble_argmax, _ = scipy_mode(per_fold_argmax, axis=1, keepdims=False)
        # build a one-hot for ensemble
        ensemble = np.zeros((probs_NKC.shape[0], probs_NKC.shape[2]))
        ensemble[np.arange(len(ensemble_argmax)), ensemble_argmax] = 1
    else:
        raise ValueError(mode)

    confidence = ensemble.max(axis=-1)
    hard_labels = ensemble.argmax(axis=-1)
    accepted = confidence >= threshold
    return accepted, hard_labels[accepted], confidence[accepted]


def save_pseudo_labels(unlabeled_paths: list[str], hard_labels: np.ndarray,
                       confidence: np.ndarray, threshold: float):
    """Save accepted pseudo-labels to CSV."""
    out = WORKDIR / "pseudo_labels"
    out.mkdir(parents=True, exist_ok=True)
    csv_path = out / "accepted.csv"
    with csv_path.open("w") as f:
        f.write("path,pseudo_label,confidence\n")
        for p, l, c in zip(unlabeled_paths, hard_labels.tolist(), confidence.tolist()):
            f.write(f"{p},{l},{c:.4f}\n")
    print(f"Saved {len(hard_labels)} accepted pseudo-labels (threshold={threshold}) to {csv_path}")


def quality_sanity_sample(unlabeled_paths: list[str], hard_labels: np.ndarray, n: int = 20) -> dict:
    """Pick n random accepted pseudo-labels for human inspection."""
    import random
    indices = random.sample(range(len(hard_labels)), min(n, len(hard_labels)))
    return {unlabeled_paths[i]: int(hard_labels[i]) for i in indices}
```

## Hard constraints

- NEVER use a fold-K model to pseudo-label fold-K's data. K-fold-safe by design — see Decision rules § K-fold safety.
- NEVER lower the confidence threshold below the model's calibration ECE. Threshold 0.9 + ECE 0.15 = predictions are actually 75% accurate at best.
- NEVER use single-threshold filtering on multi-label tasks. Use per-label thresholds.
- NEVER skip the manual quality-sanity sample (Step 5). 20 examples take 5 minutes; catching a label-flipping bug saves hours.
- NEVER claim pseudo-labels helped without re-evaluating on held-out true labels. Train metric improvement on (real + pseudo) is meaningless — it's just memorizing pseudo-labels.
